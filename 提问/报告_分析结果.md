# 分析报告：meigui.qq.com QQ登录组件不显示的原因

## 问题概述

网站 `https://meigui.qq.com/?from=platform` 在大多数现代浏览器中无法显示QQ登录组件，但在Firefox和安卓"荟萃浏览器"中可以正常显示。

## 根本原因：`document.domain` 跨域通信机制被现代Chromium浏览器废弃

### 技术细节

该网站的登录流程依赖一个**跨域iframe代理**机制来完成AJAX请求：

1. 主页面 (`meigui.qq.com`) 将请求数据存储在 `document[ajax_data_key]` 上
2. 创建一个隐藏iframe，指向 `cgi.meigui.qq.com/rose/other/meiguijqproxymeigui.html`
3. 代理iframe通过 `parent.document[ajax_data_key]` 读取请求配置
4. 代理iframe执行实际的AJAX请求，并将结果回传给父页面

这个机制的前提是：**两个不同子域名的页面都设置 `document.domain = "meigui.qq.com"`**，从而允许跨域访问彼此的DOM。

### 为什么在Chrome/Edge等浏览器中失败

Chrome 106+（2022年9月）开始强制执行 **Origin-Keyed Agent Clusters**，完全忽略 `document.domain` 的设置。控制台明确显示：

```
[WARNING] document.domain mutation is ignored because the surrounding agent cluster is origin-keyed.
```

```
SecurityError: Failed to read a named property 'document' from 'Window': 
Blocked a frame with origin "https://cgi.meigui.qq.com" from accessing a cross-origin frame.
```

这意味着代理iframe (`cgi.meigui.qq.com`) 无法访问父页面 (`meigui.qq.com`) 的 `document` 对象。

### 登录流程断裂点

```
页面加载 → checkGrayStrategy() → grayRate=100(全量命中) 
→ checkHallLogin() → onRefreshQCLoginCallback(token) 
→ AjaxTool.sendRequest("rose_login_callback", ...) 
→ jqAjax() 创建代理iframe → 代理iframe尝试读取parent.document 
→ ❌ SecurityError（静默失败）
→ success/error回调均不触发 
→ Module.Login.needLogin() 永远不被调用 
→ 登录组件不显示
```

### 为什么Firefox和荟萃浏览器可以正常工作

| 浏览器 | `document.domain` 支持 | 结果 |
|--------|------------------------|------|
| Chrome 106+ / Edge / Opera | ❌ 已废弃，强制origin-keyed | 登录组件不显示 |
| Firefox | ✅ 仍然支持（未强制origin-keyed） | 登录组件正常显示 |
| 荟萃浏览器(安卓) | ✅ 可能使用旧版WebView或自定义引擎 | 登录组件正常显示 |

Firefox至今仍允许通过设置 `document.domain` 来放宽同源策略（虽然已标记为deprecated，但未强制禁用）。荟萃浏览器作为一个小众浏览器，很可能使用了较旧的内核或未启用origin-keyed agent clusters。

## 关键代码位置

- **跨域代理函数**: `jqAjax()` — 在页面内联和 `RoseCommonCall.https.js` 中定义
- **灰度策略**: `login_strategy.js` — `grayRate: 100` 表示100%用户走QQ Connect OAuth路径
- **登录入口**: `uitab.js` 中的 `checkGrayStrategy()` 函数
- **登录组件**: `login_new.https.js` 中的 `Module.Login.needLogin()` 函数
- **失败的代理iframe**: `https://cgi.meigui.qq.com/rose/other/meiguijqproxymeigui.html`

## 结论

这不是浏览器的bug，而是网站使用了已被废弃的Web API（`document.domain` relaxation）。腾讯的玫瑰小镇网站代码较老（2014-2019年编写），未适配现代浏览器的安全策略变更。要在Chrome等浏览器中正常使用，网站需要将跨域通信机制迁移到 `postMessage` 或 CORS。

## 用户侧解决方案

如果你想在Chrome中正常使用该网站，可以尝试：
1. 使用Firefox浏览器访问
2. Chrome启动时添加参数 `--origin-agent-cluster=0`（不推荐，有安全风险）
3. 使用支持Flash的QQ浏览器高级模式（网站官方推荐）
